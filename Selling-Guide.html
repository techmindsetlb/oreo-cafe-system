1 Understanding Your Product

What Is Cafe Manager?

Cafe Manager is a complete cafe/restaurant management system that runs entirely offline on a single Windows PC. It includes:

  • POS (Point of Sale) — Take orders, split bills, print receipts
  • Order Management — View, update, and track orders in real time
  • Menu Builder — Categories and items with stock tracking
  • Inventory — Track ingredients, set minimum levels
  • Table Management — Regular tables + gaming stations (PlayStation, PC, billiards)
  • Employee Management — Roles: admin, manager, cashier, waiter, kitchen
  • Reports & Accounting — Daily/weekly/monthly sales, expenses, net profit
  • Business Day — Open/close cash register with history
  • Customer Database — Track customer info and history

How the Licensing Works

Every installation is locked to a specific computer using a hardware fingerprint (CPU, RAM, MAC addresses, motherboard UUID). A license file (.lic) is cryptographically signed with RSA-4096 (military-grade encryption) and can only be generated by you using your private key.

📋 Client runs app → 🔑 Gets Hardware ID → 📤 Sends to you → ✍️ You generate .lic → 📥 Client activates → ✅ App unlocks

What the Client Gets

  • A folder called CafeManager-Portable (15–20 MB)
  • Double-click "Start Cafe Manager.bat" to run
  • Opens in their browser at http://localhost:3001
  • No internet required (ever)
  • No Node.js installation needed
  • Database stored locally in a data/ folder

2 One-Time Setup (Your Computer)

These steps only need to be done once. After this, you can generate licenses and build distribution packages whenever you need.

Step 2.1 — Generate Your RSA Key Pair

🔐 This is the most important step. Your private key is what makes your license system secure. Anyone with your private key can forge licenses. Keep it safe.
# Navigate to the license-tool folder cd license-tool # Generate RSA-4096 key pair node generate-keys.js ✅ RSA key pair generated! private.pem → KEEP SECRET (use to sign licenses) public.pem → EMBED IN APP (use to verify licenses)

This creates two files in license-tool/:

⚠️ BACKUP YOUR PRIVATE KEY! Copy private.pem to a USB drive, cloud storage (encrypted!), or print it as a QR code. If you lose it, you cannot generate new licenses.

3 The License System Explained

How It Works (Technical)

  1. Hardware Fingerprinting — On first run, the app reads CPU model, RAM, MAC addresses, and motherboard UUID to create a unique 32-character Hardware ID.
  2. License File (.lic) — A signed JSON file containing the hardware ID, client name, expiry date, and an RSA-4096 digital signature.
  3. Signature Verification — The app uses the embedded public key to verify the signature. If the signature doesn't match, the license is rejected.
  4. Machine Locking — The hardware ID in the license must match the local machine's fingerprint. Copying a license to another PC won't work.
  5. Expiry Enforcement — The app checks the expiry date on every request. Past the date = locked out.
  6. Clock Tamper Detection — If someone moves their system clock backward, the app detects the time drift and blocks access.

Anatomy of a License File

{ // Identifies the client's machine "hardware_id": "0635f0ca89f1f5b305e48b4ce501be1b", // Your client's name (for your records) "client_name": "Joe's Cafe", // When you issued it "issued_at": "2026-06-25", // When it expires (they can't use it after this date) "expires_at": "2026-09-23", // Version (reserved for future use) "version": 1, // RSA-4096 signature — this is what prevents forgery "signature": "H9f8G7h6J5k4L3m2N1b0V9c8X7d6E5f4..." }

What Happens Without a License

  • The app starts but shows a "License Required" page
  • The client's Hardware ID is displayed prominently
  • All API endpoints return 403 Forbidden
  • The client can upload a .lic file directly through the browser
  • Once a valid license is detected, the app unlocks completely

4 Onboarding Your First Client

This is the full flow from the moment a client agrees to buy your software. Follow these steps in order.

Phase A: Client Gets the Software

  1. Zip the distribution package — Right-click dist/CafeManager-Portable/ → "Compress to ZIP" or use any archiver. The zip will be about 10–15 MB.
  2. Send it to the client — Via email, Google Drive, WeTransfer, or USB stick. The client does not need internet to use the app, but they need internet to receive the files.
  3. Client unzips — Anywhere on their PC (Desktop, C:\Program Files, etc.).
  4. Client double-clicks Start Cafe Manager.bat.
  5. A browser opens at http://localhost:3001 showing the License Required page.

Phase B: Client Gets Their Hardware ID

  1. On the activation page, the Hardware ID is shown in a box. It looks like: 0635f0ca89f1f5b305e48b4ce501be1b
  2. The client copies this ID and sends it to you via email, WhatsApp, SMS, etc.
  3. Tell the client to leave the server running.
💡 Pro tip: Ask the client to send you a screenshot of the activation page. That way you can see both the Hardware ID and confirm they reached the right page.

Phase C: You Generate the License

  1. Open a terminal/command prompt on YOUR computer.
  2. Navigate to the project's license-tool/ folder.
  3. Run the license creator:
    node create-license.js
  4. Enter the details when prompted:
    ╔══════════════════════════════════════════╗ ║ Cafe Manager - License Creator ║ ╚══════════════════════════════════════════╝ Enter hardware ID from client: 0635f0ca89f1f5b305e48b4ce501be1b Enter client name (e.g. "Joe's Cafe"): Joe's Cafe Expiry (date like "2026-12-31" or days like "30"): 90 ✅ License created: Joe_s_Cafe.lic Client: Joe's Cafe Hardware: 0635f0ca89f1f5b305e48b4ce501be1b Expires: 2026-09-23 File: /path/to/Joe_s_Cafe.lic 📤 Send this .lic file to your client.
⚠️ Expiry formats you can use:
  • 30 — 30 days from today
  • 90 — 90 days from today
  • 365 — 1 year from today
  • 2027-12-31 — Specific date (YYYY-MM-DD format)

Phase D: Client Activates

  1. Send the .lic file to the client (email, WhatsApp).
  2. The client has two ways to activate:

    Method 1: Through the browser (easiest)

    • On the activation page, click "Choose license file (.lic)"
    • Select the .lic file from their computer
    • The page shows "✅ License activated! Refresh the page to start."
    • Refresh the page → The full Cafe Manager app loads!

    Method 2: Drop it in the folder

    • Place the .lic file directly inside the CafeManager-Portable/ folder
    • Restart the app (close the command window and double-click the .bat again)

Phase E: First Use

  1. The app shows a login page.
  2. Default credentials:
    Email: admin@cafe.com Password: admin123
  3. Tell the client to change the admin password immediately! Go to Employees page → Edit the admin user → Change password.
  4. Guide the client through initial setup:
    • Go to Menu → review/add menu items and prices
    • Go to Tables → configure their table layout
    • Go to Employees → add staff accounts
    • Go to Inventory → set up stock items
    • Go to Business Day → open the first business day

Renewal Workflow

When a client's license is approaching expiry or has expired:

  1. Client contacts you — They see "License expired on [date]" when trying to use the app. They email or message you.
  2. Client sends their Hardware ID — It's still displayed on the expired license page. (The hardware ID doesn't change, so you can also keep it in your records.)
  3. You verify payment — Check that they've paid for the renewal period.
  4. You generate a new license — Same process as before:
    cd license-tool node create-license.js # Enter same Hardware ID, same client name, new expiry
  5. Send the new .lic file — The client replaces the old one in their app folder and restarts the app.
💡 Keep a client spreadsheet: Track each client's name, hardware ID, license file name, issue date, and expiry date. When they contact you for renewal, you already have their hardware ID ready.

Handling Non-Payment

When a client doesn't renew:

  • Don't send a new license file — Simple as that. The old one expires on its own.
  • The app becomes unusable (shows "License Required" page).
  • The client's data remains intact in the database. If they pay later, a new license will restore access with all their data.
  • There's no way for the client to bypass the expiry without a signed license from you.

Offering a Trial

Want to let clients try before they buy?

  • Generate a license with a short expiry like 7 (7 days) or 14 (14 days)
  • Send it to the prospect. They activate it just like a paid license.
  • When it expires, they either buy or lose access.
  • No need to change anything in the app — the license system handles it all.

6 Building & Distributing Updates

When You Release an Update

When you make improvements or fix bugs, you need to rebuild the distribution package and send it to your clients.

  1. Make your changes — Edit the source code in backend/ and frontend/.
  2. Rebuild the frontend:
    cd frontend npm run build
  3. Rebuild the distribution package:
    cd .. (back to project root) node scripts/package-dist.js
  4. Send the new folder to your clients (or just the changed files if you want to be selective).
⚠️ Client data is NOT affected by updates. The database is stored in backend/database/cafe.db (inside their app folder). As long as the client replaces the backend files but keeps the database, all their data is safe. Always tell clients to backup their database before updating!

What to Update: Full vs. Partial

You changed…Send client…
Backend code (routes, license, server) Entire backend/ folder (but keep database)
Frontend (UI, pages, styles) Entire frontend/build/ folder
Both Entire distribution package (fresh install)
💡 For minor updates: You can just ZIP the new dist/CafeManager-Portable/ folder and send it. The client unzips it somewhere new, copies their license.lic and backend/database/cafe.db from the old install, and they're good to go.

7 Anti-Piracy & Code Protection

The Protection Layers

Your software has multiple layers of protection. Each layer makes it harder for someone to steal or resell your work.

LayerWhat It DoesBypass Difficulty
1. Code Obfuscation Scrambles variable names, flattens control flow, strings are encrypted. Makes the source code unreadable. ⭐⭐ Medium — Determined hackers can reverse it
2. RSA-4096 Signatures License files must be digitally signed by your private key. Forging a signature is computationally infeasible. ⭐⭐⭐⭐⭐ Impossible — Would take centuries to crack
3. Hardware Locking License only works on the specific machine it was issued for. Can't copy licenses between computers. ⭐⭐⭐⭐ Very hard — Would need to spoof hardware IDs
4. Clock Tamper Detection Detects if someone moves their system clock backward to extend the license. ⭐⭐⭐ Hard — Can be bypassed by freezing the clock before starting the app
5. .exe Compilation (optional) Compiles everything into a single .exe. Much harder to extract code from a compiled binary than from plain .js files. ⭐⭐⭐⭐ Very hard — but not impossible
📌 The Important Truth: No offline software is 100% piracy-proof. If it runs on someone else's machine, they technically have the code. The goal is to make stealing it harder and more expensive than just paying you. For a cafe owner paying $29/month, hiring someone to crack the software would cost way more.

What Can't Be Protected Against (And That's OK)

  • System clock manipulation — A determined user could freeze their clock or use VM snapshots. The anti-tamper detection makes casual cheating difficult.
  • Memory dumping — Running code is decrypted in memory. A skilled hacker could dump and analyze it.
  • Redistribution by a trusted client — If someone buys a legitimate license and shares their copy with a friend, the friend won't have the right hardware ID. The license won't work.

Optional: Build a Single .exe

For extra protection, you can compile the app into a single Windows executable. This bundles Node.js + all code into one .exe file. The build script is provided but may need adjustments depending on your system.

# Build using the @yao-pkg/pkg fork (better compatibility) node scripts/build-exe.js

The output goes to dist/CafeManager.exe. You then distribute just the .exe + .lic file instead of the whole folder.

8 Client Support & Troubleshooting

Common Issues Clients Encounter

Issue: "Node.js is not recognized" error when running the .bat

Cause: The client doesn't have Node.js installed on their system. The current distribution package expects Node.js to be available.

Solution: Ask them to download and install Node.js from nodejs.org (the LTS version, ~25 MB installer). After installation, restart the .bat file. Alternatively, you can bundle a portable Node.js in the distribution for a fully self-contained package.

Issue: Browser shows blank page or "Cannot connect"

Solution: Check that the command window is still open (closing it stops the server). Make sure they're visiting exactly http://localhost:3001 (not HTTPS, not a different port). If the port 3001 is already in use, close other programs and try again.

Issue: "License required" page keeps showing after placing .lic file

Solution: Make sure the file is named exactly license.lic (not license (1).lic or Joe_s_Cafe.lic). The app automatically detects any .lic file, but the upload method in the browser is more reliable. Try uploading through the browser instead.

Issue: "License expired" on a new computer

Solution: The license is locked to the original machine's hardware ID. If the client gets a new computer, they need a new license. Your process: ask for the new Hardware ID → generate a new .lic → send it to them.

Issue: "System clock tampering detected"

Solution: The client's computer date/time was changed. Ask them to:

  1. Right-click the clock in the taskbar → "Adjust date/time"
  2. Turn on "Set time automatically"
  3. Click "Sync now"
  4. Restart Cafe Manager
If the issue persists, delete the data/.license-check file in the app folder and restart.

Issue: Accidentally deleted the database

Solution: Check the backend/backups/ folder. The app creates automatic daily backups at 4 AM (keeps last 30 backups). To restore, replace the cafe.db file with a backup.

Support Workflow (For You)

  1. Client reports issue — Ask for a screenshot and description.
  2. Check the server log — The command window shows real-time logs. Ask the client to copy any error messages they see.
  3. Common fixes first — Restart the app, check the .lic file, check the system clock, reinstall the app (keep the database).
  4. If nothing works — Ask the client to send you their backend/database/cafe.db file. You can run it locally to diagnose issues.
  5. For critical bugs — Fix the issue, rebuild the distribution, send them an update.

Quick Troubleshooting Checklist

  1. ☐ Is the command window still open? (Closing it stops the server)
  2. ☐ Is the .lic file in the right folder? (Should be in the same folder as the .bat file)
  3. ☐ Is the .lic file named correctly? (Any name works, but must end in .lic)
  4. ☐ Is the system clock set correctly? (Wrong date = "expired" or "tampering detected")
  5. ☐ Is there a previous .lic file interfering? (Delete old ones, keep only the latest)
  6. ☐ Has the hardware changed? (New computer = new license needed)
  7. ☐ Restart the app (close and reopen the .bat file)
  8. ☐ Restart the computer
